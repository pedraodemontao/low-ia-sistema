# Swipe Library — o cérebro que compõe

Biblioteca **global** (cross-oferta) de inteligência de copy/oferta. Cresce a cada espionagem e a cada oferta que roda. **Quanto mais você usa o sistema, mais esperto ele fica.**

## Os 2 níveis
- **`observado.md`** — o que o MERCADO usa (extraído pelo `espiao` de cada concorrente). Cru, não provado. Cresce a cada `/espionar`.
- **`validado.md`** ★ — o que converteu NO TEU funil (provado pela Cakto via `analista-funil`). Vale muito mais. É o ouro.

## O loop
1. `/espionar <url>` → o `espiao` **acrescenta** o que achou em `observado.md` (1 bloco por concorrente: nicho, fonte, data, hooks, mecanismo, ângulo, oferta, gatilhos).
2. Você roda a oferta.
3. `/analisar-funil` → quando um hook/ângulo/criativo **paga** (Cakto), o `analista-funil` **promove** pra `validado.md` (com o número que provou).
4. Próxima oferta: a skill `copy` lê `validado.md` PRIMEIRO → começa do vencedor.

## Como ler/usar
- `INDEX.md` — tabela de todas as ofertas espionadas (nicho, fonte, data).
- Filtrar por **nicho**: ao construir oferta no nicho X, puxe os blocos de nicho X (e adjacentes) de `observado` + `validado`.
- Regra: **validado > observado**. Nunca clonar literal (cair no leilão) — usar como matéria-prima pra diferenciar.

## Arquivos
- `INDEX.md` — registro de espionagens.
- `observado.md` — inteligência de mercado (por nicho).
- `validado.md` — provado pelo teu dinheiro (por nicho).
